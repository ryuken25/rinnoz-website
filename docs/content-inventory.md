# RinnOZ Content Inventory (Phase 1 — 2026-07-10)

## Stack and deployment
- Framework: Next.js App Router (not raw HTML).
- Primary page: `src/app/page.tsx`; all public content is one composed route.
- Existing order flow: `src/components/OrderModal.tsx` → client-side `mailto:` generated by `src/lib/mailto.ts`.

## Public content → proposed CMS storage
| Content | Current source | CMS model |
|---|---|---|
| Hero name, headline, subcopy, CTA labels | `Hero.tsx`, i18n dictionaries | `site_settings` |
| Artist bio/status/social links | Hero/SocialCards/content files | `site_settings` |
| Terms accepted/not accepted | `TermsSection.tsx`, dictionaries | `site_settings` |
| Anime/chibi pricing, dual USD/IDR, descriptions | `content/pricing.ts`, `PricingSection.tsx` | `packages` |
| Artwork gallery, tags, source metadata, alt text | `content/artworks.ts`, gallery/story/browse | `images` |
| Five story beat copy and tags | `i18n/dictionaries.ts`, `ArtworkStory.tsx` | `story_steps` |
| Process milestones | `ProcessTimeline.tsx`, dictionaries | `site_settings.process_steps` |
| FAQ pairs | `FaqSection.tsx`, dictionaries | `site_settings.faq` |
| Footer/CTA/copyright | `Footer.tsx` | `site_settings` |
| Order requests | `OrderModal.tsx` | `order_requests` |

## Order flow fields to preserve
- name, email, preferred contact + contact link
- payment method, style/type, character count, background, usage, deadline
- character description, lore, design, references, notes
- language, source URL, terms acceptance, honeypot `website`

## Image handling
- Current source already uses `next/image` in Hero, gallery, story, browse, pricing, and order preview.
- Existing static assets/public files should migrate to `rinnoz-media` when CMS is implemented; do not regress existing fallback data until DB public fetch is verified.
